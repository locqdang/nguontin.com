# Quickstart: NguonTin MVP

1. Start from `spec.md` for the approved MVP scope and user-story priorities.
2. Use `plan.md` for architecture boundaries, implementation strategy, and verification approach.
3. Use `data-model.md` before backend schema, ownership, enum, or visibility work.
4. Use `tasks.md` as the execution checklist for setup, foundations, and user-story delivery.
5. Add `contracts/` artifacts only when a later planning pass needs explicit API or interface contracts.
